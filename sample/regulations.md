# 情報セキュリティ規定

## 第1章 認証・アクセス制御

### 1.1 認証方式の要件

すべてのシステムは、利用者を一意に識別できる認証機構を備えなければならない。
共有アカウントの使用は原則として禁止する。

認証方式は以下のいずれかを採用すること:
- パスワード認証（12文字以上、複雑性要件あり）
- SSO（SAML 2.0 または OpenID Connect）
- 証明書認証

### 1.2 多要素認証の適用基準

以下の条件に該当するシステムには、多要素認証（MFA）を必須とする:
- 機密（confidential）または制限（restricted）データを扱うシステム
- インターネットからアクセス可能なシステム
- 管理者権限を有するアカウント

ただし、社内ネットワークからのみアクセス可能で、かつ公開（public）データのみを
扱うシステムについては、MFA を免除できるものとする。

### 1.3 アクセス制御

最小権限の原則に基づき、利用者には業務遂行に必要な最小限の権限のみを付与すること。
権限の付与・変更・削除は、申請と承認のプロセスを経ること。
退職者・異動者のアカウントは、人事異動日から5営業日以内に無効化すること。

## 第2章 データ保護

### 2.1 データ分類

すべてのデータは以下の4段階に分類すること:
- **公開（public）**: 公開情報。保護要件なし
- **社内（internal）**: 社内利用に限定。外部への意図しない流出を防止すること
- **機密（confidential）**: 個人情報、契約情報、技術情報等。暗号化必須
- **制限（restricted）**: 経営機密、特別な個人情報等。暗号化およびアクセスログ必須

### 2.2 暗号化要件

機密（confidential）以上のデータを扱うシステムは、以下の暗号化を実施すること:
- **保存時（at-rest）**: AES-256 以上の暗号化
- **通信時（in-transit）**: TLS 1.2 以上

制限（restricted）データについては、保存時と通信時の両方の暗号化を必須とする。

ただし、保存時の暗号化については、データがクラウドサービスプロバイダの
マネージド暗号化（例: AWS SSE-S3, GCP デフォルト暗号化）で保護されている場合、
これを代替手段として認めるものとする。

### 2.3 データ保持と削除

データの保持期間は、法令要件または業務要件に基づき定めること。
保持期間を超過したデータは、復元不能な方法で削除すること。
個人情報の保持期間は、利用目的の達成に必要な期間を超えてはならない。

## 第3章 外部サービス利用

### 3.1 SaaS 利用の審査基準

外部SaaSサービスを業務利用する場合、以下の項目を審査すること:
- サービスプロバイダのセキュリティ認証（SOC 2, ISO 27001 等）の有無
- データ保存先のリージョン（日本国内またはGDPR適合国に限定）
- サービスレベル契約（SLA）の内容（稼働率 99.5% 以上を推奨）
- 契約終了時のデータエクスポートおよび削除手順

### 3.2 クラウドインフラの利用基準

IaaS/PaaS を利用する場合、以下を遵守すること:
- 利用するリージョンを事前に申告し、承認を得ること
- ネットワーク構成（VPC、セキュリティグループ等）を文書化すること
- 管理コンソールへのアクセスに MFA を適用すること
- 利用料金の監視体制を整備すること

### 3.3 API連携のセキュリティ

外部サービスとAPI連携を行う場合:
- API キーおよびトークンは環境変数またはシークレット管理サービスで管理すること
- ソースコードへの直接埋め込みは禁止する
- API 通信は TLS 1.2 以上で暗号化すること
- レート制限およびタイムアウトを設定すること
